# b) How did you fix the tp-link-decrypt program?

I first ran the script as it is out of the box.

I followed these commands:

$ git clone https://github.com/robbins/tp-link-decrypt

$ cd tp-link-decrypt

$ ./preinstall.sh        # Install dependencies

$ ./extract_keys.sh      # Extract RSA keys from TP-Link's GPL code

$ make

$ bin/tp-link-decrypt Tapo_C200_firmware.bin

I think I got it to work just by following the commands that were presented in the blog, I did not have to do any changes to the code.

But what happaned and what does that script actually do? 

tp-link-decrypt — a tool that extracts RSA/AES keys from TP-Link's own GPL source releases, then decrypts the firmware. The tool:
  
  1. Parses the firmware header to identify the encryption scheme
  2. Uses the extracted RSA-2048 private key to unwrap the AES session key
  3. Decrypts the firmware body with that key

# d) What findings did you make?
The temp_password contained this information: 

iVar1 = atoi(pcVar3 + 1);       // uptime when password was created

  iVar4 = get_sys_uptime();        // current uptime
  
  if (iVar4 <= iVar1 + 0x1c20) {  // 0x1c20 = 7200 seconds = 2 hours
  
  The temp password is only valid for exactly 2 hours after it was created, measured in system uptime (not wall clock
  
  time — so it can't be fooled by changing the clock).
